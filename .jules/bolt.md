## 2025-05-15 - [Optimize UploadFile Memory Usage and Resizing Speed]
**Learning:** For high-throughput image processing APIs using FastAPI, using `await file.read()` loads the entire file into RAM, which can quickly lead to memory exhaustion with large files (e.g., 100MB limit). A more efficient pattern is to use the underlying file stream (`file.file`) with `seek(0, 2)` and `tell()` for size validation and passing the stream directly to `PIL.Image.open()`. Additionally, `Image.BICUBIC` offers a significant speed boost over `Image.LANCZOS` for resizing operations with minimal impact on output quality.
**Action:** Always validate `UploadFile` size using file stream seeking and use the stream directly for processing to avoid redundant memory copies. Default to `BICUBIC` resampling unless the highest possible quality is explicitly required.
